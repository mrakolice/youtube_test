# Назначение проекта

Это тестовый проект, представляющий из себя упрощенную копию сервиса загрузки видео и работы с ними, а именно:
1. Сама загрузка видео
2. Просмотр
3. Лайки
4. Авторизация

Каждый из пунктов - отдельный микросервис. Общение между микросервисами происходит либо через кафку для асинхронных операций
либо по JWT-токену, который генерируется на стороне авторизационного сервиса и хранится между сессиями.

У каждого микросервиса одинаковая структура:
```
|-- app/
|   |-- api
|       |-- requests
|       |-- responses
|       |-- views
|   |-- core
|       |-- config
|       |-- security
|   |-- db
|       |-- base
|       |-- session
|   |-- models
|       |-- __init__.py
|       |-- <model_name>.py
|-- migrations
|-- tests
```
Каждая модель в отдельном файле. Каждая модель подгружается в __init__.py, чтобы в дальнейшем в микросервисе 
можно было использовать абсолютный импорт `from app import models` и в коде `models.ModelClass1`, `models.ModelClass2`, 
а не `from api.models.model_name import ModelClass1` и так далее.
